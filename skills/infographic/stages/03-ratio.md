# 阶段 3：比例选择（Aspect Ratio Selection）

**目标：** 根据用户需求自动推荐或让用户选择合适的图片比例。

---

## 四种比例选项（按使用频率排序）

### 1️⃣ 16:9（最常用）
- **描述**：横版宽屏
- **用途**：PPT 展示、电脑端全屏幕展示、大屏显示
- **场景示例**：会议演示、网页展示、桌面背景
- **特点**：横向空间充足、适合放置复杂内容

### 2️⃣ 3:4（竖版中等）
- **描述**：竖版，宽度:高度 = 3:4
- **用途**：小红书、抖音、微博等社交媒体发布
- **场景示例**：社交媒体内容、竖屏手机浏览但不全屏
- **特点**：竖向较长、适合强调核心信息点

### 3️⃣ 9:16（竖版长条）
- **描述**：竖版超长屏
- **用途**：移动端手机全屏幕展示、Story、Reel 等竖屏形式
- **场景示例**：手机全屏、短视频背景、App 广告
- **特点**：竖向拉长、突出竖屏体验

### 4️⃣ 1:1（正方形）
- **描述**：完全正方形
- **用途**：通用方形格式、适配多种平台
- **场景示例**：头像、缩略图、方形卡片、通用分享
- **特点**：均衡对称、无方向偏好

---

## 你需要做的事

### 第一步：解析用户需求
在用户的原始需求中寻找以下关键词或场景描述：

| 关键词/场景 | 自动推荐比例 |
|---|---|
| "PPT" / "演示" / "展示" / "电脑" / "大屏" | **16:9** |
| "小红书" / "抖音" / "微博" / "社交" / "社媒" | **3:4** |
| "手机" / "竖屏" / "全屏" / "Story" / "Reel" | **9:16** |
| "头像" / "方形" / "卡片" / "缩略图" / "通用" | **1:1** |

### 第二步：判断与反馈

**情况 A：用户需求中明确指定了比例或使用场景**
- 📌 **自动适配**：无需询问，直接确认选定的比例
- 例子：用户说"我要在 PPT 上展示" → 自动选 16:9
- 例子：用户说"这个我想发小红书" → 自动选 3:4
- 确认语：**"好的，根据你的需求，已为你选择 [比例] 格式"**

**情况 B：用户需求中没有明确指定比例或场景**
- 🤔 **询问用户**：展示 4 种选项让用户选择
- 展示顺序按使用频率：16:9 → 3:4 → 9:16 → 1:1
- 可以提示"推荐 16:9，也可选其他"
- 让用户选择其中一个

---

## 本阶段输出物

### DoD（Definition of Done）

- 确认了**用户选择的图片比例**（16:9 / 3:4 / 9:16 / 1:1）
- 我向用户复述确认：
  - 自动适配情况："根据你的需求（[场景]），已为你选择 **[比例]**"
  - 用户选择情况："很好，为你的信息图采用 **[比例]** 格式"
- 一旦确认，进入**阶段 4**（提示词生成）

---

## 提示词传递规则

将选定的比例以下列方式传递到提示词生成阶段：
- **16:9**：`aspect ratio 16:9 landscape`
- **3:4**：`aspect ratio 3:4 portrait`
- **9:16**：`aspect ratio 9:16 portrait`
- **1:1**：`aspect ratio 1:1 square`

在提示词中确保清晰表达所选的比例约束。
