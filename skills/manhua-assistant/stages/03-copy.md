# 阶段3：文案定稿（Copy Spec：唯一真值）

**目标：** 把内容变成"上图文案规格表（Copy Spec）"：逐字定稿 + 字数预算 + 格子结构。阶段4只负责"封装成提示词"，不再改文案本身。

## 压缩规则（默认规则）

- 先删掉"讲解用"的背景句，只保留"结论/动作"
- 默认：每格 **1 行短句**，每行尽量 **≤10 个字**
- 用直观词替代术语堆叠：更稳/更可控/更省心/能跑通/能复刻…

## 隐喻与元素

参考 `references/visual-metaphors.md` 获取分类隐喻库。核心原则：选择最直观、最大众的隐喻，避免抽象术语。

常用隐喻快索引：
- **超载/复杂** → 塞爆的背包、打结毛线
- **失败→成功** → 角色表情递进（困惑→思考→行动→喜悦）
- **工具/技能** → 工具箱、积木堆叠
- **对比冲突** → 左灰暗拥挤 / 右明亮整洁

## 角色一致描述（如果阶段2设计了角色）

- **主角形象**：（复述阶段2的角色设定）
- **跨格一致性**：所有格子中，主角的发型/衣着/配色完全一致
- **表情变化**：根据格子情绪递进设计表情（如：困惑→思考→行动→喜悦）

## Copy Spec 输出模板（你交付给用户确认的格式）

每份 Copy Spec 对应【一张多宫格漫画】（一次生成一张图，图内包含多个格子）：

```
图编号：图1（一张多宫格漫画）
格子数量：N 格
画幅：1:1 / 3:4 / 2:1（根据格子数量选择）
布局：手绘散落 / 左右错落 / 斜向阶梯 / 自然散落…（格子位置刻意不对齐，像手绘草稿）
文字预算：标题1行 + 每格1行短句；禁小字；禁模型加字
角色设定（如有）：
  主角：……（一句话描述）

每格文案（均为同一张图内的不同格子）：
标题：……
格1：（画面描述 + 文字）
格2：（画面描述 + 文字）
格3：（画面描述 + 文字）
……
```

注意：一份 Copy Spec 最终生成【一张图片】，这张图片内包含多个宫格（格1、格2、格3…）。如需多张独立漫画，请分别输出多份 Copy Spec。

## 本阶段输出物

- **Copy Spec（逐字定稿）**：用户确认后才进入阶段4
- 若用户说"字还是多"：优先删到"每格 1 句或仅关键词"
