# 阶段1：需求澄清

**目标：** 在"拆内容/做漫画"之前，先把用户的真实需求说清楚：**内容是什么、用在什么场景、谁来看、想要几宫格、是否需要固定角色**。其它默认值（如画幅/风格）不打扰用户。

## 你需要问用户的 4 个问题（优先级从高到低）

1. **内容（先给我原文/要点）**：把要做成漫画的内容贴过来（整段/小节/提纲都行）。
   - （可选补一句）你希望读者看完**记住哪一句话/得到什么结论**？
2. **场景（用在哪、怎么看）**：手机近看 / PPT 投影远看 / 海报打印 / 公众号头图？
   - 如果多个都有：**哪个是主场景**（我优先为它优化字号与格子数量）？
3. **受众（谁来看）**：小白 / 从业者 / 老板 / 学生…（决定措辞与隐喻）
4. **几宫格 + 角色**：
   - 你希望用几宫格？（2格/3格/4格/6格/9格…）
   - 需要固定角色吗？（例如：同一个主角贯穿所有格子 / 每格不同角色 / 不需要角色）

## 本阶段输出物

### DoD（Definition of Done：没确认不进阶段2）

- 你复述给用户确认的 1 句话：
  - **内容主题 + 主场景 + 受众 + 几宫格 + 角色需求 + 画幅建议**
- 默认补齐但不打扰用户：画幅根据格子数量自动推荐、默认风格（奶油纸 + 彩铅水彩 + 轻涂鸦）
- 需求小结（不超过 5 条）：
  - 例如：`手机近看为主`、`给小白看`、`4格手绘散落布局漫画`、`格子位置不对齐`、`同一主角贯穿`、`画幅1:1`

## 给用户的参考输入（让他照抄也行）

- 内容：……（粘贴原文/要点/提纲）
- 场景：手机近看为主（也会用于 PPT）
- 受众：给……看（小白/从业者/老板/学生…）
- 宫格：几格？布局偏好？（2格横/3格/4格田字/6格/9格…）
- 角色：需要固定角色吗？（同一主角贯穿 / 每格独立 / 不需要）
- 目标：希望读者看完记住……
