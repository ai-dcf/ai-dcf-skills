# 基础提示词模板（Base Prompt）

这是生成提示词的骨架，支持**智能信息量分析**和**动态结构生成**。

## 模板特性

✨ **智能分析** - 系统自动分析用户输入的复杂度和信息量
✨ **动态结构** - 根据内容自动调整内容点数和深度
✨ **灵活结论** - 根据需要动态添加或省略结论部分

## 模板结构

### 基础框架
```
# 画幅 & 尺寸
Create a 16:9 landscape infographic image

# 标题与副标题
Title (large, bold): {标题}
Subtitle (small): {副标题 - 可选}

# 主体内容结构
Main content layout: {版式类型}
[根据复杂度动态生成的内容结构...]

# 结论部分（可选 - 根据内容自动判断是否需要）
Bottom conclusion/CTA: {结论 - 仅当必要时包含}

# 风格描述
[风格块 - 见对应的 style-*.md]

# 强制约束
Strict requirements:
- Chinese text must be clear, readable, large font
- No extra text, no watermarks, no signatures
- Sufficient whitespace, clear alignment
- No cluttered background, no random characters
- Do not add any text other than specified above
```

## 智能分析机制

### 系统如何分析信息量

1. **文本长度分析** - 评估主题描述的详细程度
2. **关键词识别** - 检测复杂度指示词和信息密度指示词
3. **内容类型判定** - 识别流程、对比、列表、时间线等
4. **复杂度评级** - 分为 simple / medium / complex
5. **信息密度评级** - 分为 low / medium / high

### 根据分析结果动态调整

| 分析结果 | 调整项 | 建议值范围 |
|--------|--------|---------|
| **简单** | 内容点数 | 2-3 个 |
| | 内容深度 | 浅显，1-2 句说明 |
| | 结论部分 | 通常不需要 |
| **中等** | 内容点数 | 3-4 个 |
| | 内容深度 | 中等，1-2 句说明 |
| | 结论部分 | 如有明确关键点则需要 |
| **复杂** | 内容点数 | 5-6 个 |
| | 内容深度 | 深层，2-3 句说明 |
| | 结论部分 | 通常需要总结 |
| **高信息密度** | 内容点数 | +1 额外点 |
| | 每点详度 | 增加详细程度 |
| | 结论部分 | 更可能需要 |

### 结论部分判定规则

系统根据以下条件自动判定是否需要结论：

```
IF 用户提供了关键点 → 通常需要结论
   - 类型：insight（洞察）或 cta（行动号召）

ELSE IF 包含"应该", "建议", "需要"等CTA关键词 → 需要CTA
   - 类型：cta（行动号召）

ELSE IF 复杂度为复杂 → 需要结论
   - 类型：summary（总结）

ELSE → 不需要结论
   - 信息图可自然结束，无需额外收尾
```

## 填充指南

### 用户可提供的信息

| 参数 | 示例 | 用途 |
|-----|------|------|
| **主题** | "AI 的 5 大风险" | 必需，用于分析复杂度 |
| **受众** | "小白/从业者" | 可选，调整表述方式 |
| **关键点** | "了解风险，更好拥抱 AI" | 可选，触发结论部分 |

### 系统自动生成的内容

| 部分 | 说明 |
|-----|------|
| **内容点数** | 根据复杂度自动确定（2-8 个） |
| **内容深度** | 根据信息密度调整（浅/中/深） |
| **版式选择** | 根据内容类型推荐（流程/对比/列表等） |
| **结论部分** | 根据需要自动添加或省略 |

### 典型场景

**场景 1：简单概念介绍**
```
主题：什么是区块链
→ 系统判定：简单
→ 内容点数：3 个
→ 结论部分：不需要
```

**场景 2：复杂流程分解**
```
主题：一个完整的数字营销漏斗分析和优化策略
→ 系统判定：复杂，高信息密度
→ 内容点数：6 个（5 + 1）
→ 内容深度：深层
→ 结论部分：需要（类型：summary）
```

**场景 3：明确的行动导向**
```
主题：中小企业应该如何选择云服务供应商
关键点：做出正确选择的5大标准
→ 系统判定：中等复杂度，明确的关键点
→ 内容点数：5 个
→ 结论部分：需要（类型：cta）
```

## 版式类型速查

系统根据内容自动选择最合适的版式：

| 版式 | 适用场景 | 内容组织方式 |
|-----|--------|------------|
| **流程图** | 步骤、阶段、周期 | 阶段间用箭头连接，从上到下或左到右 |
| **对比图** | 对比、区别、差异 | 左右或上下两列，突出差异 |
| **卡片列表** | 要点、类型、特点 | 并列展示的卡片或区块 |
| **时间线** | 历史、演变、发展 | 时间顺序排列的里程碑 |
| **综合版** | 通用主题 | 中心概念 + 周围支撑点 |
