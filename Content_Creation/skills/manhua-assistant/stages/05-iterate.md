# 阶段5：迭代润色（让漫画更好看）

**目标：** 根据用户反馈快速迭代：更趣味、更少字、更清楚、更连贯。

## 常见反馈 → 对应动作

| 用户反馈 | 对应动作 |
|---------|---------|
| "字太多/太丑" | 每格压到 **1 句或仅关键词**，删掉副标题或把副标题变短 |
| "角色不统一/变了样" | 在提示词里加角色一致性约束：**同一角色在所有格子中发型/衣着/配色完全一致** |
| "看不懂在讲什么" | 检查格子间的叙事连贯性，增加视觉连接（角色延续、场景连续、动作承接） |
| "隐喻看不懂" | 换成更大众隐喻（毛线/背包/路牌/工具箱） |
| "格子对齐太完美/像信息图" | 强调"格子位置刻意不对齐，左右上下自由错落，像手绘草稿般随意自然" |
| "格子边框歪斜倾斜" | 强调"格子边框保持水平垂直"（但允许格子位置自然错位） |
| "布局太死板机械" | 允许格子大小灵活变化、位置自然错位、间距有节奏感，营造手绘感 |
| "缺少手绘感/太整齐" | 加入"手绘草稿感"描述；允许分镜线轻微波动；留白形状不规则 |
| "中文不清楚" | 强调"大字号/无小字/干净字体/避免花体" |
| "模型乱加字/补充说明" | 在提示词里加硬约束：**除指定文字外不要生成任何额外文字** |
| "风格不对/太像日漫" | 强调风格块为唯一基础，加负面约束：**不要扁平矢量/不要3D/不要摄影/不要日式漫画风** |
| "颜色太乱" | 强调色调协调：全图主色调一致，格子间颜色和谐过渡 |

## 回归检查

迭代后对照 `templates/checklist.md` 过一遍再交付。遇到不确定的问题，参考 `references/common-issues.md`。确保：
- [ ] 格子数量明确，布局方式已指定
- [ ] 叙事连贯：格子顺序与故事发展逻辑一致
- [ ] 角色与风格统一：同一角色跨格保持一致
- [ ] 文字可读：每格1行短句或关键词
- [ ] 视觉重点：每格有明确焦点
